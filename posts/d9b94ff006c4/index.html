Kotlin中使用注解

注解

定义在类、方法、字段 、参数前的”注释“。注解（annotation）可以被编译器打包进class文件。

元注解

有一些注解可以修饰其他注解，这些注解就称为元注解（meta annotation）。

常用 元注解有：

  • @Retention：指定该注解是否存储在编译后的 class 文件中，以及它在运行时能否通过反射可见 （默认都是 true）；
  • @Target：指定可以用该注解标注的元素的可能的类型（类、函数、属性、表达式等）；
  • @Repeatable：允许在单个元素上多次使用相同的该注解；
  • @MustBeDocumented：指定该注解是公有 API 的一部分，并且应该包含在生成的 API 文档中显示的类或方法的签名中。

运行时注解

运行时注解@Retention(AnnotationRetention.RUNTIME)，大多数时候是在运行时使用反射来实现所需效果。

Java可以使用反射API读取Annotation的方法有：

  • X.isAnnotationPresent(Class)：判断某个注解是否存在于X
  • X.getAnnotation(Class)：读取Annotation

编译时注解

注解处理器（annotation processor）是javac的一个工具，它用来在编译时扫描和处理注解。你可以自定义注解，并注册相应的注解处理器，用于处理你的注解逻辑。

在项目结构中创建Java or Kotlin Library类型的模块，在其中实现一个自定义注解处理器（AbstractProcessor），并重写process方法。随后你就可以在process方法内对注解进行处理。
